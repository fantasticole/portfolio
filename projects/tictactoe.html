Tic-Tac-Toe:

Code | Site

This is the first project I built from scratch for my application to Hacker School Recurse Center. It is a two-player Tic-Tac-Toe game that tracks how many wins each player has and alternates which player starts each game.


Challenge: Check for the winner efficiently each time a move is made.

Solution: When an available tile is clicked, function checkWin() gets called. This runs function getPlayerMoves(), which returns an array of the current players squares. This array is checked against all possible winning combinations. If there is a match, the setWin() function is called, which turns off pointer events for all tiles using CSS.


Challenge: Stop play once there is a victor or stalemate.

Solution: Each tile has an id number. When a tile is available, a function called setMove() pushes an object containing the currentPlayer and the box's id to an array called boxes. Each time a tile is clicked, a function called isSquareAvailable is called, which checks to see if that tile's id is in boxes. If not, setMove() is called. Otherwise nothing happens. Each time setMove() is called, the function checkWin() is called as well. This function checks an array of the current player's boxes against an array of winning combinations. If any one of those combinations is found in the current player's boxes, setWin() is called. Once a player wins or there is no way to win and no more moves can be made, the visitors can reset the board by clicking "Play again!" underneath the game board.


Challenge: Track the amount of wins each player has.

Solution: I used a JavaScript Object called tally to store the amount of wins each player has. When a visitor first arrives at the page, tally.forX and tally.forO are both set to 0. Once the checkWin function returns true, the setWin function gets called with the currentPlayer passed in as a parameter. setWin increases the appropriate player's tally based on the input.


Challenge: Alternate player turn in the game and player starting each game.

Solution: I created a JavaScript variable currentPlayer, which is originally set to "X", who always starts the first game. I also created a function called otherPlayer, which which takes a player as a parameter and then returns the opponent. function otherPlayer(player){
var mapping = {"X": "O", "O": "X"};
return mapping[player];
};
. Each time a player moves, currentPlayer. is set to otherPlayer(currentPlayer). Once the game is over, the resetBoard function is called, which figures out whose turn it is by adding tally.forX and tally.forO. If the number is even, X will start the next game. If it's odd, O starts. This means that in the case of a draw, nobody wins and the same player starts the next game.
